Last reviewed: 2026-04-18

## Summary
- Nursing programs often prioritize reliability and exam/proctoring compatibility over high-end performance.
- If your program uses proctored exam software, OS compatibility matters more than CPU/GPU speed.

## Common software (examples)

### ExamSoft (Examplify)
- Purpose: high-stakes exams/proctoring (school-controlled).
- Key risk: strict OS/support policies; may block VMs/remote desktop.
- Best next step: confirm your school’s exact exam tool name + version + allowed OS list.

### Respondus LockDown Browser
- Purpose: proctored testing inside a browser.
- Key risk: can be picky about OS versions, security settings, and screen sharing tools.
- Best next step: confirm whether your school uses Respondus Monitor (webcam) in addition to LockDown Browser.

### Microsoft 365 (Word/Excel/PowerPoint)
- Generally available on Windows and macOS.
- Used for papers, presentations, and basic coursework.

### Video calls and LMS
- Typical: Zoom/Teams and Canvas/Blackboard/D2L.
- Usually fine on Windows or macOS (but exam/proctoring tools may be stricter).

## Choosing Mac vs Windows (how to decide)

Ask these first (in this order):
1) What exam/proctoring software does your program require (exact name)?
2) Does the school explicitly allow macOS for exams?
3) Do they forbid virtual machines, remote desktop, or dual-boot setups for exams?

General guidance:
- If exam/proctoring compatibility is uncertain, Windows is usually the safer choice.
- If the school explicitly supports macOS for required exam software, Mac can be fine.

## What to ask the student (follow-ups)
- “Do you know the exact exam/proctoring tool name (Examplify vs Respondus vs something else)?”
- “Is your school requiring Windows-only software?”
- “Will you be taking proctored exams on your laptop?”
