# Software Notes

Put software compatibility notes here (requirements, OS limitations, licensing, known issues).

## How to write good software notes

- One software per file: `knowledge/software/<software-key-or-name>.md`
- Include version when it matters (example: “Respondus LockDown Browser vX.Y”).
- Prefer short headings and bullet lists (better retrieval).
- If you’re not sure, write it as a question + what to verify (don’t guess silently).
- Add links under a “Sources” section (the bot won’t show them to end users, but they help you maintain accuracy).

## Suggested structure (copy/paste)

```md
# Software: <Name>

## Compatibility
- OS: Windows/macOS/Any (be explicit)
- Notes: (VM allowed? remote desktop works? admin rights needed?)

## Minimum requirements (if you have them)
- RAM:
- Storage:
- CPU:
- GPU:

## What to ask the user (if unclear)
- (example) “Are you on Windows or macOS?”
- (example) “Is this basic use or heavy use?”

## Sources
- <official vendor link>
```
